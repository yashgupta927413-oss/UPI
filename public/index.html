🚀 End-to-End API Integration Guide

Follow these 5 simple steps to integrate the P2P UPI Payment Gateway into your custom website, app, or billing system:

Step 1: Initiate Transaction (Backend-to-Gateway)
When a customer checks out, your server calls the POST /api/external/pay/initiate endpoint with the amount and a unique order ID. The gateway automatically chooses a VPA from the pool, applies paise-level fingerprinting (e.g., ₹500.03 instead of ₹500.00), and returns a unique checkoutUrl.
Step 2: Redirect Customer (Frontend-to-Checkout)
Redirect the customer to the returned checkoutUrl in their browser. They will see your customized branding logo, payment timer, and a dynamic QR code for scan-and-pay, or a "Pay Now" deep link button if they are on a mobile device.
Step 3: Device SMS Forwarding (Mobile Alert Sync)
When the customer pays, your bank sends a transaction credit SMS alert to your Android phone. The SMS forwarder app on your phone posts the SMS to your specific VPA webhook URL: POST /api/pay/sms-webhook/:userId/:assignedUpi?token=YOUR_API_KEY.
Step 4: Auto-Reconciliation & Settlement
The gateway matches the SMS amount and VPA against active pending transactions. It resolves and approves the order, updates pool capacity, and releases the paise fraction for subsequent orders.
Step 5: Receive Instant Webhook Notification (Gateway-to-Backend)
The gateway dispatches a secure JSON payload to the Merchant Webhook Endpoint configured in your Settings, notifying your server of the successful checkout payment so you can deliver the digital goods/services instantly.

🔑 Authentication Headers

We support two auth header standards. Set either of these headers to your raw upi_live_... API key:

Authorization: Bearer YOUR_API_KEY
Or use:
X-API-Key: YOUR_API_KEY

📡 1. Initiate Payment Transaction

Send a POST request to initiate a new checkout routing window:

POST http://localhost:3000/api/external/pay/initiate
Content-Type: application/json

{
  "orderId": "order_1001",
  "amount": 499.00
}

Expected JSON Response (201 Created):

{
    "success": true,
    "orderId": "order_1001",
    "baseAmount": 499,
    "finalAmount": 499.03, // Unique fingerprint resolved
    "assignedUpi": "merchant.savings@sbi",
    "upiString": "upi://pay?pa=merchant.savings%40...",
    "checkoutUrl": "https://gateway.com/checkout/order_1001",
    "expiresAt": "2026-07-06T17:19:02Z",
    "status": "PENDING"
  }

📈 2. Check Payment Status

Query the status of an order using a simple GET endpoint:

GET http://localhost:3000/api/external/pay/status/order_1001

Expected JSON Response (200 OK):

{
    "success": true,
    "orderId": "order_1001",
    "baseAmount": 499,
    "finalAmount": 499.03,
    "status": "APPROVED", // APPROVED, PENDING, or CANCELLED
    "expiresAt": "2026-07-06T17:19:02Z",
    "utrNumber": "123456789012",
    "createdAt": "2026-07-06T17:12:02Z"
  }

How to configure SMS forwarding:

  1. Install an SMS Gateway / Forwarder app on the Android phone containing the SIM card registered to the UPI ID.
  2. Configure the app to forward incoming SMS messages.
  3. Set the destination/target URL to the copied URL above.
  4. Set the HTTP method to POST.
  5. The message payload should be sent under field key body or message or text.
